## 1. 기획 의도 및 전략 (The 'Why')

```

프로젝트 명: 2030 청년을 위한 정책, 쓱쓱 웹페이지 개발

제작 목적

- 여러 곳에 흩어져 있는 2030 청년 맞춤형 정부 및 지자체 지원 정책을 한곳에 모아 제공하는 통합 플랫폼 구축
- 복잡하고 어려운 정책 정보를 사용자가 쉽고 빠르게 탐색할 수 있도록 직관적인 환경 제공

타겟 관객

- 주거, 취업, 금융, 교육 등 실생활에 밀접한 지원 정책 정보가 필요한 20대~30대 청년층
- 자신에게 맞는 혜택을 찾는 데 어려움을 겪거나 정보가 부족한 청년들

기대 성과
- 청년들의 정책 접근성 향상 및 실질적인 혜택 수혜율 증가
- 맞춤형 정보 제공을 통한 정책 탐색 시간 단축 및 정보 비대칭 해소

```

```
너가 필요한 컷이나 디자인, 이런걸 다 정해서 필요한 영상컷이 어떻게 되고, 어떤 흐름으로 영상이 진행되는지 다 작성해봐

```

```
협업 포인트: 우선 처음에 gpt나 gemini가 주는 대로 컷별로 프롬프트를 실행했지만, 
영상 자체는 달라도 받아들이는 입장에서는 반복적이거나, 
루즈하다고 느껴지는 구간이 있어서 그런 부분은 추가로 수정했습니다. (3번이 기존 프롬프트 / 4번 파일이 최종 프롬프트입니다.)

```

```

협업 포인트: 정책 어플 광고 영상이지만,  인물의 감정 또한 드러나는 영상이기에, 어느 정도의 감정 서사가 필요했고, 긴 영상이 아니기에 초반은 조용하게,이후엔 분위기가 밝아지는 느낌을 원했습니다.
```

## 3. 프롬프트 엔지니어링 성과 (Before & After)

초기에 입력했던 프롬프트와, 결과가 마음에 들지 않아 개선한 최종 프롬프트를 비교하여 작성해 주세요.

```

AI에게 컷 별로 영상 프롬프트를 적어달라고 하고, 그대로 생성해보았지만 절반이상 제가 생각한 것과 맞지 않았습니다.
현장 촬영팀으로 영상을 찍을 때, 그리고 ai를 이용해서 영상을 생성했을 때 중요하다고 생각한 건 영상 내에서의 '디테일'이였습니다.
전체적으로 영상의 흐름이 어느정도는 맞아야 하고, 앞뒤 영상이 서로 이어져야 영상이라고 할 수 있다고 생각합니다.

초기 프롬프트는 장면의 시각적 연출을 중심으로 구성되어 있었으며,
스마트폰 → 키워드 → 카드 등장으로 이어지는 단계적 설명 구조를 가지고 있었습니다. 
그래서 해당 구간이 영상 자체는 다르지만 영상으로써 전달되는 메세지가 반복되고, 루즈해지는 느낌을 받았습니다.

최종 프롬프트에서는 3컷으로 나뉘어진 해당 구간을 1컷으로 통합했습니다. 
'rise → organize → transform → align'과 같은 구조 중심 동사를 사용하여 혼란 상태가 정리되는 과정을 그려나가 보았고,
또한 'controlled motion', 'no chaotic movement'와 같이 모델의 움직임을 통제하는 지시어를 추가함으로써 
연출 중심 묘사에서 메시지 중심 설계로 개선했습니다.

그 결과, 장면 수는 감소했으나 서사 밀도와 전달력이 향상되었으며, 반복적 표현이 제거되어 루즈한 느낌은 지울 수 있었습니다.


```
